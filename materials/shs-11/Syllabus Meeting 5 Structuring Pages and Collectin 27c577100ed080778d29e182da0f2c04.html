Topic 1: Structuring Content with <div> and Layout Elements

The <div> Element: The All-Purpose Container

The <div> element is one of the most common tags in HTML. Think of it as an empty cardboard box that you can use to group other elements together. It has no special look on its own, but it's incredibly useful for organizing your page into sections so you can style them or move them around together.

  • <div> is a "Block" Element: By default, a <div> is a block-level element. This means it always starts on a new line and takes up the full width available on the page, like a whole paragraph block.

Example: <div> as a container In this example, we group a heading and two paragraphs inside a single <div>. We can then add a border to the entire group with one style attribute.

<div style="border:1px solid black; padding:10px; margin-bottom:10px;">
  <h2>London</h2>
  <p>London is the capital city of England.</p>
  <p>London has over 9 million inhabitants.</p>
</div>

Result:

London

London is the capital city of England. London has over 9 million inhabitants.

How to Center a <div> If you give a <div> a specific width that is less than the full page width, it will align to the left by default. To center it, you can use the inline style margin:auto;.

<div style="width:300px; margin:auto; border:1px solid black; padding:10px;">
  <h2>London</h2>
  <p>London is the capital city of England.</p>
  <p>London has over 9 million inhabitants.</p>
</div>

This tells the browser to automatically calculate equal margins on the left and right, pushing the box into the center.

HTML Layout Elements: Giving Your Page Meaning

While you can build an entire website using only <div>s, modern HTML gives us special tags that work like <div>s but have meaningful names. These are called semantic elements because they describe the meaning or purpose of the content inside them.

Why use Semantic Elements?

  • Accessibility: Screen readers can use them to help visually impaired users navigate your page.
  • SEO: Search engines like Google can better understand your content, which can help your site rank higher.
  • Clarity: It makes your code easier for you and other developers to read.

Meet the Layout Team:

  • <header>: The introduction for a page or a section. Usually contains the logo, site title, and main navigation.
  • <nav>: Specifically for a block of navigation links.
  • <section>: A major, standalone section of a document. Think of it like a chapter in a book.
  • <article>: For self-contained content that could be distributed on its own, like a blog post, news story, or forum post.
  • <aside>: For content that is indirectly related to the main content, like a sidebar with related links or an author bio.
  • <footer>: The footer for a page or a section. Usually contains copyright information, contact details, and links.
  • <details> & <summary>: A fun interactive widget that the user can open and close to see more details. <summary>provides the visible heading.

Example of a Semantic Layout: Here is an example of a simple webpage layout using these meaningful elements.

<body>

<header style="background-color:#f1f1f1; text-align:center; padding:20px;">
  <h1>My Awesome Website</h1>
</header>

<nav style="background-color:#333; overflow:hidden;">
  <a href="#" style="color:white; padding:14px 20px; float:left;">Home</a>
  <a href="#" style="color:white; padding:14px 20px; float:left;">About</a>
  <a href="#" style="color:white; padding:14px 20px; float:left;">Contact</a>
</nav>

<section style="padding:20px;">
  <h2>About Me</h2>
  <p>This is where I talk about myself.</p>
  <details>
    <summary>More Fun Facts</summary>
    <p>I also enjoy playing the guitar and coding!</p>
  </details>
</section>

<footer style="background-color:#333; color:white; text-align:center; padding:10px; margin-top:20px;">
  <p>Copyright 2025</p>
</footer>

</body>

Topic 2: Collecting Information with HTML Forms

An HTML <form> is used to collect input from the user. This is how you create login pages, contact forms, search bars, and quizzes!

The Core Form Elements

  • <form>: This is the main container for all your form elements. It has two crucial attributes:
    • action: This attribute tells the browser where to send the form data when it's submitted. This is usually a URL to a script on a server.
    • method: This attribute tells the browser how to send the data. The two most common methods are GET and POST.

Understanding GET vs. POST

Think of it like sending mail:

  • method="GET" is like sending a postcard. ✉️ The data you submit is attached to the URL in the address bar, visible to everyone. It looks like this: .../process.html?name=John&age=14. This method is fine for things like search queries, but you should never use it for sensitive information like passwords.
  • method="POST" is like sending a sealed letter. 📮 The data is sent in the body of the web request, hidden from the URL. This is more secure and is used for sending private information, like when you log in to an account.
  • <input>: This is the most used form element. The type attribute changes how it looks and behaves.
  • <label>: This defines a label for an input element. It's important for two reasons:
    1. It helps screen-reader users understand what the input field is for.
    1. It allows users to click on the label text to focus on or select the input field, which is great for small radio buttons or checkboxes.
    • The for attribute of the <label> must match the id attribute of the <input> to connect them.

Common Input Types

TypeDescription
<input type="text">A single-line text input field.
<input type="radio">A radio button (for selecting ONE of many choices).
<input type="checkbox">A checkbox (for selecting ZERO or MORE of many choices).
<input type="submit">A button that submits the form.
<input type="button">A simple clickable button.
The name Attribute is VITAL! Notice that each input field must have a name attribute to be submitted. This name becomes the "key" for the data. For example, name="username" with a value of "Sarah" becomes username=Sarah when it's sent. If the name attribute is missing, the data from that field will not be sent.

Topic 3: Showcase: Seeing Form Data in Action!

Let's do a quick experiment to see how the GET method works. This is a sneak peek for when we learn JavaScript later!

Activity: The Data Postcard

  1. Create two new HTML files in the same folder: formindex.html and process.html.
  1. Copy the code below into each file.

File 1: formindex.html

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>HTML Form</title>
</head>
<body style="font-family: sans-serif; padding: 20px;">

    <h1>Enter Your Details</h1>

    <!--
      The <form> tag is the container for our inputs.
      - The 'action' attribute tells the browser to send the data to our 'process.html' page.
      - The 'method="GET"' attribute tells the browser to send the form data in the URL.
        You will see it in the address bar on the next page!
    -->
    <form action="process.html" method="GET">

        <!-- Label for the name input. The 'for' attribute links it to the input's 'id'. -->
        <label for="name">Name:</label>
        <br>
        <!--
          The 'name' attribute is very important. It becomes the key for the data we send.
          For example: name=John
        -->
        <input type="text" id="name" name="name" required>
        <br><br>

        <!-- Label for the age input. -->
        <label for="age">Age:</label>
        <br>
        <!-- The 'name' attribute here will be 'age'. For example: age=25 -->
        <input type="number" id="age" name="age" required>
        <br><br>

        <!-- A simple submit button to send the form. -->
        <button type="submit">Submit</button>

    </form>

</body>
</html>

File 2: process.html

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Data Sent</title>
</head>
<body style="font-family: sans-serif; padding: 20px;">

    <h1>Data Sent Successfully!</h1>

    <!--
      With pure HTML, we cannot dynamically read the data from the URL.
      However, you can look at the browser's address bar right now!
      You will see the data you entered, like this:
      .../process.html?name=John&age=14
    -->
    <p style="background-color: #eee; border: 1px solid #ccc; padding: 10px; border-radius: 5px;">
        <strong>Teaching Moment:</strong> Check the browser's address bar! You'll see the values you just entered attached to the URL. That's the `GET` method in action.
    </p>

    <h2>Data We Imagine Was Saved:</h2>
    <p>We can't show your *actual* data here without JavaScript, but imagine we saved it!</p>
    <p>name = John</p>
    <p>age = 14</p>

    <br>

    <!-- This is a link that takes the user back to the form page. -->
    <a href="formindex.html">Go Back</a>

</body>
</html>

Instructions:

  1. Open formindex.html in your browser.
  1. Fill in your name and an age, then click "Submit".
  1. Look at the address bar on the process.html page! You will see your data right there in the URL. You've just used the GET method!

Topic 4: More Awesome Form Elements

  • <textarea>: A multi-line text box for longer messages.
    • rows: Specifies the visible number of lines.
    • cols: Specifies the visible width (in characters).
    <textarea name="message" rows="5" cols="30" style="width:250px; height:100px;">
The cat was playing in the garden.
</textarea>
  • <select> and <option>: A drop-down list. <select> is the container and <option> is each choice.
    • selected: Add this attribute to an option to make it the default choice.
    • size: Specifies the number of visible options (creates a scroll box).
    • multiple: Allows the user to select more than one option (usually by holding Ctrl or Cmd).
    <label for="cars">Choose a car:</label>
<select id="cars" name="cars">
  <option value="volvo">Volvo</option>
  <option value="saab">Saab</option>
  <option value="fiat" selected>Fiat</option>
  <option value="audi">Audi</option>
</select>
  • <button>: A clickable button. Always specify the type attribute. type="button" is a clickable button, while type="submit"will submit the form.
    <button type="button">Click Me!</button>
  • <fieldset> and <legend>: Use <fieldset> to draw a box around related data in a form, and <legend> to give the group a caption. This is great for organization.
    <form action="/action_page.php">
  <fieldset>
    <legend>Personalia:</legend>
    <label for="fname">First name:</label><br>
    <input type="text" id="fname" name="fname" value="John"><br>
    <label for="lname">Last name:</label><br>
    <input type="text" id="lname" name="lname" value="Doe"><br><br>
    <input type="submit" value="Submit">
  </fieldset>
</form>
